# 2026 世界杯 PWA Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a React + Vite + TypeScript PWA for a 2026 World Cup data app matching the approved UI demo and product decisions.

**Architecture:** The app is a client-side PWA with feature modules for matches, standings, players, teams, news, and watch links. ESPN/public data is normalized into internal types, cached locally, and backed by an embedded snapshot; manual override JSON files provide Chinese labels, watch/highlight links, news links, and missing player metadata.

**Tech Stack:** React, Vite, TypeScript, Vitest, Testing Library, vite-plugin-pwa, CSS modules or plain feature CSS following the demo style.

---

## File Structure

- Create `worldcup-2026-pwa/` as the formal app project.
- Create `src/app/` for shell layout, navigation, routing state, and PWA status.
- Create `src/features/matches/` for schedule cards, match details, watch/highlight links, and knockout bracket preview.
- Create `src/features/standings/` for 12 group standings and best-third ranking.
- Create `src/features/players/` for Top 20 leaderboards and player profiles.
- Create `src/features/teams/` for team profile views.
- Create `src/features/news/` for title/source/link news cards.
- Create `src/services/` for API clients, cache, snapshots, and data normalization.
- Create `src/data/overrides/` for manually maintained watch links, highlight links, news links, names, flags, and player fallbacks.

## Acceptance Criteria

- The formal app matches the approved demo direction.
- Every match card can open match details and has per-match live/highlight actions.
- All 12 groups and the best-third ranking are visible.
- Player leaderboards show Top 20 for the selected stat only.
- Missing player avatars render attractive fallbacks.
- News links work as external links and do not copy third-party full text by default.
- Mobile and desktop layouts have no page-level horizontal overflow.
